## 你的任务

1. 从原始发现中筛选出最真实、最有价值的痛点
2. 丢弃以下内容：
   - 与已有痛点本质相同或高度相似的
   - 证据不足或疑似编造的
   - 太泛泛没有具体场景的
3. 对保留的每个痛点，深度回答五个问题
4. 最多保留 3 个，可以少于 3 个，质量不够就不要凑

## 五个分析维度
1. **谁在痛** — 具体人群、职业、规模、特征
2. **什么场景下痛** — 具体做什么事时遇到问题
3. **有多痛** — 频率、时间/金钱成本、情绪强度
4. **现在怎么忍的** — 替代方案及其不足
5. **为什么忍不下去了** — 近期加剧因素

## 输出格式

严格只输出 JSON，不要有其他文字：

